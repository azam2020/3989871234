Issue Report: Slow Network Throughput on VF

Issue Summary

HSD No.: 3989871234

Title: Slow Network Throughput Observed on Virtual Function

Description: A noticeable slowdown in network throughput was observed on a Virtual Function (VF) assigned to a virtual machine. The Parent Function (PF) performance is unaffected, but traffic over the VF experiences significant latency and packet loss.

Hardware Configuration

  • Hardware Model: Lenovo ThinkSystem SR650
  • Network Adapter: Mellanox ConnectX-4
  • Number of Network Interfaces: 6 (SR-IOV enabled)
  • Memory: 256 GB RAM
  • CPU: AMD EPYC 7502P (2x)
  • Firmware Version: 12.20.1000

Software Configuration

  • Operating System: RHEL 8.5
  • Kernel Version: 4.18.0-305.12.1.el8.x86_64
  • Network Driver Version: mlx5_core 5.1.0
  • SR-IOV Configuration:
    • PFs: 2
    • VFs per PF: 16
  • Network Configuration Tool: nmcli 1.28.0
  • Virtualization Software: KVM 7.9.0
  • Network Management: systemd-networkd 247

Steps to Reproduce

  1. Enable SR-IOV in BIOS/UEFI and configure 16 VFs on each PF.
  2. Assign a VF to a virtual machine running on KVM.
  3. Run a network performance benchmark (e.g., iperf3) between two VMs, one using the PF and the other using the VF.
  4. Measure the throughput and observe the network latency.
  5. Reboot both the VM and host system and repeat the performance test.

Expected Result

The network throughput on the VF should be similar to that of the PF, with minimal latency and no packet loss.

Actual Result

Network throughput on the VF is significantly lower than the PF, with high latency and packet loss.
